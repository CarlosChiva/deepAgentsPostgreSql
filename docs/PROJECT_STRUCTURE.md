## File-by-file Descriptions

### Root Level

| File | Purpose |
|------|---------|
| `docker-compose.yml` | Defines two services: `postgres` (PostgreSQL 16-alpine with pgvector) and `chat-api` (FastAPI built on Python 3.12 + uv). Declares shared network, volumes, health checks, and depends_on. |
| `.env.example` | Template with all env vars: `POSTGRES_USER`, `POSTGRES_PASSWORD`, `POSTGRES_DB`, `POSTGRES_HOST`, `LLM_MODEL`, `OPENAI_API_KEY` or `ANTHROPIC_API_KEY`, `LANGSMITH_API_KEY`, `LANGSMITH_PROJECT`. |
| `.env.production` | Actual secrets (add to `.gitignore`). |

### `services/chat-api/`

| File | Purpose |
|------|---------|
| `pyproject.toml` | Project definition: name, Python 3.12+, [build-system] using uv/setuptools, [tool.uv.sources], and [dependency] list. |
| `uv.lock` | Generated lockfile by uv sync. |
| `Dockerfile` | Multi-stage: (1) `python:3.12-slim` builder with uv to install deps, (2) `python:3.12-slim` runtime with only the production venv. |
| `.dockerignore` | Excludes `.env`, `.venv`, `__pycache__`, `.git`, `tests/` from build context. |
| `app/main.py` | FastAPI app: creates the app instance with OpenAPI docs, startup event (initialize agent), and includes the chat router. |
| `app/config.py` | Pydantic `Settings` class reading from env vars (PostgreSQL URI, LLM provider keys, model name, thread TTL). |
| `app/agent.py` | **Core AI logic**. Factory function `get_agent()` that: (1) creates a PostgresSaver from the DB URI and calls `.setup()`, (2) instantiates `create_deep_agent()` with the LLM model, system prompt, default tools, and the checkpointer. |
| `app/database.py` | PostgreSQL connection management. Uses `psycopg` connection pool for the checkpointer. Provides a helper that formats the `POSTGRES_HOST`, `POSTGRES_PASSWORD`, `POSTGRES_DB` into a connection string. |
| `app/routers/chat.py` | FastAPI router with prefix `/chat`, tag `"chat"`. Two endpoints: `POST /` for sending messages, `GET /{thread_id}` for retrieving history. |
| `app/models/chat.py` | Pydantic models for request/response: `ChatRequest` (thread_id: str, message: str), `ChatResponse` (thread_id: str, reply: str, timestamp: datetime), `ThreadStatus` (thread_id, message_count, last_message). |
| `app/models/history.py` | Pydantic models for history retrieval: `MessageEntry` (role, content, timestamp), `HistoryResponse` (thread_id: str, messages: list[MessageEntry]). |
| `app/services/chat_service.py` | **Business logic layer**. Two functions: `send_message()` (invokes DeepAgent with thread_id, streams response via SSE), and `get_thread_history()` (reads checkpoint states from PostgresSaver for a thread_id and reconstructs the message sequence). |

### `infrastructure/postgres/`

| File | Purpose |
|------|---------|
| `Dockerfile` | FROM `postgres:16-alpine`. Installs `pgvector` extension for future embedding support. Sets locale and default collation. |
| `init/01-init.sql` | Optional DDL (indexes, custom schemas). Can be empty or omitted — PostgreSQL's default schema is sufficient. |
